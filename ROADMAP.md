# Roadmap

This roadmap describes the first public direction for Aegora. It is intentionally conservative while the fork identity, governance, and release process are being established.

## Phase 0: Foundation

Goal: make the project safe, understandable, and contributor-ready.

- Preserve license and attribution
- Establish Aegora project identity
- Add governance and mentorship docs
- Improve contribution guidance
- Create issue and pull request templates
- Define initial release and stability expectations
- Complete the public and internal Aegora naming migration
- Prepare JSR publishing configuration and release guidance

## Phase 1: Contracts and Documentation

Goal: make Aegora excellent for documented, reliable APIs.

- Add structured OpenAPI 3.1 export API
- Improve route introspection documentation
- Add examples for schema, validation, and response contracts
- Create a plugin authoring guide
- Add production recipes for common backend patterns
- Document JSR-first installation and publishing workflows

## Phase 2: Developer Experience

Goal: make Aegora easier to adopt and easier to debug.

- Improve framework error messages
- Add official testing helpers
- Add starter app workflow
- Add examples for auth, CORS, static files, WebSockets, and deployment
- Explore migration helpers from Elysia to Aegora

## Phase 3: Ecosystem

Goal: grow a reliable plugin and contributor ecosystem.

- Define official plugin quality criteria
- Create plugin templates
- Create package naming conventions
- Publish mentored plugin issues
- Document long-term support expectations
- Establish JSR package publishing policy for official Aegora packages

## Non-Goals for the Foundation Phase

- Rewriting framework internals immediately
- Breaking public APIs without migration guidance
- Removing required upstream attribution
- Positioning Aegora as a hostile fork

## Guiding Principles

- Stability over novelty
- Clear contracts over magic
- Respectful review over gatekeeping
- Mentorship over sink-or-swim contribution
- Public design discussion over surprise rewrites
