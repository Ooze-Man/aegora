# Security Policy

Aegora is in an early foundation phase. Security reporting and response practices will become more formal as the maintainer group grows.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Until a dedicated security contact is established, open a private GitHub security advisory for this repository when available.

A good report includes:

- Affected version or commit
- Clear reproduction steps
- Expected impact
- Whether the issue is already public
- Suggested fix, if known

## Response Goals

Maintainers should aim to:

- Acknowledge valid reports promptly
- Confirm impact before public disclosure
- Prepare a fix and release notes
- Credit reporters when they want credit

## Supported Versions

Aegora has not reached a stable release yet. Until then, the supported version is the current default branch and latest pre-release package, if published.
